# ナレッジベース（RAG用）

このディレクトリに配置した Markdown ファイル（`*.md`）は、議事録作成時に自動的に参照されます。

## 使い方

1. **Markdown ファイルを作成**
   - `terms.md`: 用語集
   - `project_info.md`: プロジェクト情報
   - その他任意の `.md` ファイル

2. **内容を記述**
   - 見出し（`#`）でセクション分けすると、検索精度が向上します
   - 会社名、プロジェクト名、技術用語、略語などを記載

3. **自動的に反映**
   - ファイルを保存するだけで、次回の議事録作成時から参照されます
   - キャッシュは自動的に更新されます

## ファイル例

### terms.md（用語集）

```markdown
# プロジェクト名

## 社内プロジェクト
- Project Alpha: 新製品開発プロジェクト
- Project Beta: 既存システムの刷新

## 技術用語
- RAG: Retrieval-Augmented Generation（検索拡張生成）
- LLM: Large Language Model（大規模言語モデル）
```

### project_info.md（プロジェクト情報）

```markdown
# チーム構成

## 開発チーム
- リーダー: 田中さん
- フロントエンド: 佐藤さん、鈴木さん
- バックエンド: 高橋さん

# 定例会議

## 週次ミーティング
- 毎週月曜日 10:00〜
- 進捗報告と課題共有
```

## ベストプラクティス

- **具体的に書く**: 「新プロジェクト」より「Project Alpha（新製品開発）」
- **略語を展開**: 「RAG（検索拡張生成）」のように正式名称も記載
- **定期的に更新**: プロジェクト情報は最新に保つ
- **セクション分け**: 見出しを使って構造化する

## キャッシュについて

- 埋め込みベクトルは `.rag_cache/embeddings.json` にキャッシュされます
- ファイルが更新されると自動的に再計算されます
- キャッシュが壊れた場合は `.rag_cache/` ディレクトリを削除してください
